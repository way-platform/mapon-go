Custom_targets

List Custom Targets

GET https://mapon.com/api/v1/custom_targets/list.json
  • Description
  • Parameters
  • Response
  • Error codes
Retrieves a list of custom targets defined for the company. These targets can represent various measurable parameters, thresholds, or goals that can be tracked.
key *
string
API Key

The response contains a data array, which holds a list of custom target objects. Each target object includes the following fields:

  • id (int) - The unique identifier for the custom target.
  • label (string) - A human-readable name or description for the custom target.
  • units (string) - The unit of measurement relevant to this target (e.g., "minutes", "celsius", "km", "revolutions_per_minute").
{
    "data": [
        {
            "id": 593,
            "label": "Engine Overheat Threshold",
            "units": "celsius"
        },
        {
            "id": 594,
            "label": "Max Idle Time",
            "units": "minutes"
        },
        {
            "id": 595,
            "label": "Service Interval",
            "units": "km"
        }
    ]
}

Get Custom Target Values

GET https://mapon.com/api/v1/custom_targets/values.json
  • Description
  • Parameters
  • Response
  • Error codes
Retrieves historical values for a specified custom target for one or more units (vehicles/assets) within a given date-time range. The period for which data can be requested is limited to 31 days.
key *
string
API Key
unit_id *
int || []
One or more unit IDs
target_id *
int
The unique integer identifier (sent as a string) of the custom target for which to retrieve values. This parameter is required.
from *
datetime
Start of the date-time range (inclusive). Required. ISO8601 format (e.g., '2025-05-01T00:00:00Z' or '2025-05-01T00:00:00+03:00').
till *
datetime
End of the date-time range (inclusive). Required. ISO8601 format. Must be after 'from'.

The response is an object containing a single data field. The data field is an object where each key is a string representation of a unit_id for which values were requested.

Each unit_id key maps to an array of value objects. If a requested unit has no data for the specified target and period, its corresponding array will be empty.

Each value object in the array contains:

  • target_id (int) - The identifier of the custom target type these values belong to (this will match the target_id request parameter).
  • value (string|number|boolean) - The recorded value of the custom target. The actual data type can vary depending on the target's nature (e.g., "Active", 123.45, true).
  • gmt (string) - The UTC timestamp indicating when the value was recorded, in ISO8601 format with a "Z" (Zulu) suffix (e.g., "YYYY-MM-DDTHH:MM:SSZ").
{
    "data": {
        "533384": [
            {
                "target_id": 101,
                "value": "Active",
                "gmt": "2025-05-14T10:00:00Z"
            },
            {
                "target_id": 101,
                "value": "Inactive",
                "gmt": "2025-05-14T11:09:55Z"
            }
        ],
        "533385": [
            {
                "target_id": 101,
                "value": "Active",
                "gmt": "2025-05-14T09:30:00Z"
            }
        ],
        "533386": []
    }
}
