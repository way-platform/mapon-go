Vehicle_pin_auth

Create Device Mappings

POST https://mapon.com/api/v1/vehicle_pin_auth/device_mappings_create.json
  • Description
  • Parameters
  • Response
  • Error codes
Create device mappings between vehicles and devices for PIN authentication
key *
string
API Key
mappings *
array
Array of device mapping objects
unitId *
int
Vehicle/Unit ID
deviceId *
string
Device ID (max 255 characters)
Returns batch operation results with summary statistics and individual results for each mapping.
summary - contains total count, successful count, and failed count of operations.
results - array of individual operation results, each containing status (success/failed), unitId, deviceId, and optional message for failed operations.
{
    "data": {
        "summary": {
            "total": 2,
            "successful": 1,
            "failed": 1
        },
        "results": [
            {
                "status": "success",
                "unitId": 123,
                "deviceId": "device-001"
            },
            {
                "status": "failed",
                "unitId": 124,
                "deviceId": "device-002",
                "message": "Device mapping already exists"
            }
        ]
    }
}
CodeDescription
1Missing mappings parameter
2Invalid mappings parameter - must be non-empty array
3Invalid mapping format - each mapping must be an object
4Missing unitId parameter in mapping
5Invalid unitId parameter - must be positive integer
6Missing deviceId parameter in mapping
7Invalid deviceId parameter - must be non-empty string with max 255 characters

  • Description
  • Parameters
  • Response
  • Error codes
Create PIN codes for drivers
key *
string
API Key
pins *
array
Array of driver PIN objects
driverId *
int
Driver ID
pin *
string
PIN code (exactly 4 numeric digits)
Returns batch operation results with summary statistics and individual results for each PIN creation.
summary - contains total count, successful count, and failed count of operations.
results - array of individual operation results, each containing status (success/failed), driverId, and optional message for failed operations.
{
    "data": {
        "summary": {
            "total": 2,
            "successful": 1,
            "failed": 1
        },
        "results": [
            {
                "status": "success",
                "driverId": 456
            },
            {
                "status": "failed",
                "driverId": 457,
                "message": "Driver with existing PIN"
            }
        ]
    }
}
CodeDescription
8Missing pins parameter
9Invalid pins parameter - must be non-empty array
10Invalid PIN format - must be exactly 4 numeric digits
12Missing driverId parameter in PIN object
13Invalid driverId parameter - must be positive integer
14Missing pin parameter in PIN object
