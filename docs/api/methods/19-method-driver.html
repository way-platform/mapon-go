Get driver activity intervals (daily) | BETA

GET https://mapon.com/api/v1/driver/daily_activities.json
  • Description
  • Parameters
  • Response
  • Error codes

BETA VERSION This end-point is fully usable but there might be issues. Please contact client support if you encounter any problems.

Get a list of driver activity intervals grouped by days.
Grouping by days happens according to company timezone
Please note that due to technical limitations data will be selected for whole days. For example, if you pass period YYYY-MM-DD 12:00:00 - YYYY-MM-DD 20:00:00, the data will be returned for period YYYY-MM-DD 00:00:00 - YYYY-MM-DD 23:59:59 instead.
Maximum period: 31 days
Can select only a single driver
Data sources: CAN, Tachograph driver files
key *
string
API Key
from *
datetime
UTC datetime string in Y-m-dTH:i:sZ format
till *
datetime
UTC datetime string in Y-m-dTH:i:sZ format
driver *
int
Driver ID
include
string
Comma separated list of additional data to include.
Currently supported values:
  • card_events - include information about driver card insertion / removal events
  • work_place_events - include information about events when driver has started / finished a work period
Response contains array of objects. Each array element corresponds to a single day.
Properties of day object:
  • day - datetime value of the start of the day
  • summary - all values are total durations in seconds
  • activities - array of activity objects

activity object structure:
  • start - timestamp
  • end - timestamp
  • duration - seconds
  • status - status value. Possible values: DRIVING, REST, WORK, AVAILABLE, CARD_INSERTED, CARD_REMOVED, WORK_PERIOD_STARTED, WORK_PERIOD_FINISHED
  • source - activity source identifier. Possible values: ddd - driver tachograph file, can - data obtained via device CAN connection or unkn - unknown data source. Appears only when available data does not cover the whole selected time period. In such cases gaps are filled with REST activities
  • unitId - id of the vehicle to which driver was assigned during the activity

Associate external drivers with Mapon drivers

POST https://mapon.com/api/v1/driver/associate_external_driver.json
  • Description
  • Parameters
  • Response
  • Error codes
Allows associating external drivers with Mapon drivers. The request supports both single associations and batch associations.
key *
string
API Key
associations *
array
List of associations between external drivers and Mapon drivers
driverId *
int
Id of Mapon driver who will be associated to external driver
employeeId *
string
Id of external driver who will be associated to Mapon driver. Id does not have to be numeric - it is more like a username type of identifier
Returns batch operation results with summary statistics and individual results for each mapping.
summary - contains total count, successful count, and failed count of operations.
results - array of individual operation results, each containing status (success/failed), driverId, employeeId, companyId, and optional message for failed operations.
{
    "data": {
        "summary": {
            "total": 2,

                "employeeId": "EMP001",
                "companyId": 123

            {
                "status": "failed",
                "driverId": 124,
                "employeeId": "EMP002",
                "companyId": 123,
                "message": "Driver not found"
            }
        ]
    }
}
CodeDescription
1Missing associations array
2Invalid associations array - must be non-empty array
3Missing or invalid driverId parameter
4driverId must be a positive integer
