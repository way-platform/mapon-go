Tell_tale

Get Tell Tale Values

GET https://mapon.com/api/v1/tell_tale/values.json
  • Description
  • Parameters
  • Response

Retrieves FMS tell tale values for specified vehicles within a date range. Tell tales are dashboard indicator lights and warnings from the vehicle's FMS (Fleet Management System) interface, representing real-time status of various vehicle systems including brakes, engine, transmission, lighting, emissions, and safety systems. The FMS standard defines 75 different tell tale types organized into 5 blocks, covering everything from basic indicators like parking brake and fuel level to advanced systems like lane departure warning and electric vehicle battery management. Each tell tale can report status values from 0 (Off) to 7 (Not Available), with intermediate values indicating different warning levels. The period for which data can be requested is limited to 31 days.

limit
int
Maximum number of records to return. Range: 1-10000, default: 1000

The response contains a data object where each key is a string representation of a unit_id for which values were requested.

Each unit_id key maps to an array of tell tale value objects. If a requested unit has no data for the specified period, its corresponding array will be empty.

Each tell tale value object contains:

  • name (string) - Human-readable tell tale name (e.g., "PARKING_BRAKE", "ENGINE_OIL").
  • telltale_id (int) - Numeric tell tale identifier (1-75).
  • value (int) - Numeric status value (0-7).
  • value_title (string) - Human-readable status description.
  • datetime (string) - The UTC timestamp when the tell tale value was recorded, in ISO8601 format with "Z" suffix.

Status Values (FmsTelltaleStatus enum):

  • 0 - OFF
  • 1 - CONDITION_RED (Critical warning)
  • 2 - CONDITION_YELLOW (Warning)
  • 3 - CONDITION_INFO (Information)
  • 4-6 - Reserved
  • 7 - NOT_AVAILABLE
