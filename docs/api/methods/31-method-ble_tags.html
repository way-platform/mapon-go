Get BLE tag list

GET https://mapon.com/api/v1/ble_tags/list.json
  • Description
  • Parameters
  • Response
Returns a list of BLE tags belonging to company. Only active tags are included.
key *
string
API Key
page
int
Page number. Default value: 1
per_page
int
Entry count to return per request. Default: 10. Min: 10. Max: 150
sort_by
string
Field name to sort by. Allowed values:beacon_id, name, radius, first_scan_gmt, last_scan_gmt, scan_count, location_updated_at, created_at
sort_order
string
Sort direction. Allowed values: ASC, DESC
All dates are returned in YYYY-MM-DDTHH:II:SSZ format. Example: 2019-04-08T12:00:00Z
location_precision_radius field value is in meters.
Fields name,location_precision_radius,first_scan,last_scan,location_changed and fields from location can be null.
location_precision_radius (by default 50m if empty) determines the radius of approximated tag location circle. If the tag is scanned outside this location, a new location will be registered.

Get BLE tag historic point data

GET https://mapon.com/api/v1/ble_tags/history_point.json
  • Description
  • Parameters
  • Response
  • Error codes
Get BLE tag historic data for single datetime point. Looks for closest (by time) values within 30 day period of the passed datetime.
key *
string
API Key
id
int
BLE Tag unique ID. Either id or beacon_id parameter is required. id takes precedence if both are provided
beacon_id
string
BLE Tag serial ID. Either id or beacon_id parameter is required. id takes precedence if both are provided
datetime *
string
Datetime string. Can be in any valid format, but preferred format is YYYY-MM-DDTHH:II:SSZ format. Example: 2019-04-08T12:00:00Z. Datetime must be in GMT timezone
includes *
array
Allowed values:
scan - returns data for the closest tag scan.
location - returns data for the closest tag location
All dates are returned in YYYY-MM-DDTHH:II:SSZ format. Example: 2019-04-08T12:00:00Z
car_id, rssi, voltage, temperature and humidity fields can be null.
voltage returns values in mV (microvolts).
temperature returns values in Celsius.
humidity returns relative humidity value as percent.
location.lat and location.lng represent the center of the approximated tag location. The approximated location is a circle with radius as specified by the tag location precision radius (or 50m default radius).
location.bounds - outer border for the location circle polygon. Returned as array of coords (lat, lng).

  • Description
  • Parameters
  • Response
  • Error codes
Returns a list of tag scans in period. Maximum period: 31 days.
key *
string
API Key
id
int
BLE Tag unique ID. Either id or beacon_id parameter is required. id takes precedence if both are provided
beacon_id
string
BLE Tag serial ID. Either id or beacon_id parameter is required. id takes precedence if both are provided
from *
string
Period start datetime string. Can be in any valid format, but preferred format is YYYY-MM-DDTHH:II:SSZ format. Example: 2019-04-08T12:00:00Z. Datetime must be in GMT timezone
till *
string
Period end datetime string. Maximum period: 31 days. Can be in any valid format, but preferred format is YYYY-MM-DDTHH:II:SSZ format. Example: 2019-04-08T12:00:00Z. Datetime must be in GMT timezone
page
int
Page number. Default value: 1
per_page
int
Entry count to return per request. Default: 10. Min: 10. Max: 150
All dates are returned in YYYY-MM-DDTHH:II:SSZ format. Example: 2019-04-08T12:00:00Z
car_id, rssi, voltage, temperature and humidity fields can be null.
voltage returns values in mV (microvolts).
temperature returns values in Celsius.
humidity returns relative humidity value as percent.
